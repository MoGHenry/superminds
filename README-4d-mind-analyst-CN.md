# 4D 思维分析师

[English](https://github.com/MoGHenry/superminds/blob/main/README-4d-mind-analyst.md) | 中文 | [skills.sh](https://skills.sh/moghenry/superminds/4d-mind-analyst)

</div>

一个智能体技能，能够利用并行智能体从四个独立视角同时分析任何问题、现象或主题。灵感源自爱德华·德博诺的平行思维方法论。

## 工作原理

每个视角作为独立的并行智能体运行，最终将所有结果综合为统一的分层输出。这种方法的优势在于：每个视角都能发现其他视角遗漏的洞察。

```
输入 → 视角选单 → 并行智能体 → 综合分析 → 分层输出
```

1. 接收用户提出的主题、问题或现象
2. 展示视角选单（默认运行全部四个视角）
3. 为每个选定视角分配一个并行智能体
4. 收集所有结果并综合为统一分析
5. 交付最终的分层输出

## 四大视角

### 1. 以用户为中心的思维

模拟受主题影响最大的个人或群体，通过具体的角色模拟来挖掘真实感受、隐忧和未明确表达的期望。

**适用场景：** 当你需要理解人的层面——谁受到了影响、他们的真实感受是什么、以及他们不会公开说出口的想法。

### 2. 产品思维

拆解底层需求，分析现有方案为何失败，并假设理想解决方案的形态。

**适用场景：** 当你需要理解问题与方案的匹配度——真正的需求是什么、已经尝试过什么、以及十倍优于现状的方案应该是什么样的。

### 3. 选题思维

分析主题的集体情感共鸣——它为何吸引关注、触动了哪根神经、以及其相关性能持续多久。

**适用场景：** 当你需要解读当下的文化脉搏——为什么这个话题此刻重要、谁最关心、它是昙花一现还是一场持久运动。

### 4. 课程思维

提炼可传授的方法论——可复制的模式、步骤拆解、新手常见障碍和课程架构。

**适用场景：** 当你需要将隐性知识显性化——哪些内容可以被教授、学习路径是什么、以及初学者通常在哪里卡住。

## 使用方法

通过请求多维度分析来触发该技能：

```
"分析 AI 编程智能体的崛起"

"拆解远程办公的反弹潮——背后到底发生了什么？"

"从不同角度审视这个问题：为什么初级开发者的岗位正在消失？"

"创作者经济倦怠趋势的深层原因是什么？"
```

你也可以选择特定视角：

```
"仅使用产品思维和用户中心思维分析 X"

"对这个主题运行视角 1 和视角 3"
```

### 输出格式

综合输出遵循分层结构：

| 章节 | 内容 | 适用人群 |
|------|------|---------|
| **执行摘要** | 跨视角核心洞察，3-5句话 | 时间有限的决策者 |
| **各视角详细分析** | 每个视角的完整分析，附有「核心洞察」标签 | 希望获取全貌的深度思考者 |
| **交叉综合分析** | 共识点、张力点、盲区、意外关联 | 寻找非显而易见洞察的战略思考者 |
| **下一步行动** | 3-5 项按优先级排列的行动建议，关联具体视角 | 需要根据分析采取行动的人 |

## 技能结构

```
4d-mind-analyst/
├── SKILL.md                    # 编排器 — 调度逻辑与视角注册表
├── references/
│   ├── user-centric.md         # 视角 1：利益相关者共情与角色模拟
│   ├── product.md              # 视角 2：需求拆解与方案假设
│   ├── topic-selection.md      # 视角 3：文化共鸣与情绪生命周期
│   ├── curriculum.md           # 视角 4：可传授方法论与课程设计
│   └── synthesis.md            # 合并逻辑 — 综合所有视角
├── docs/
│   └── plan.md                 # 实施计划与设计决策
└── README.md
```

## 核心设计原则

- **具体优于笼统** — 每个视角都指向具体的、有名有姓的专家和特定的角色原型，绝不使用"某领域专家"或"用户"这样的泛称
- **先独立，后综合** — 各视角先独立运作；真正的价值在于矛盾与共识的碰撞中揭示出单一视角无法看到的东西
- **深度匹配重要性** — 随意的提问获得简明的回应，战略级的市场分析则获得相应深度的处理
- **自由表达优于刻板模板** — 每个视角以最适合其分析的方式自然书写
- **语言灵活性** — 输出语言匹配你的输入语言，也可以明确指定

## 添加新视角

该技能设计支持扩展。添加新视角只需：

1. 按照现有模式创建 `references/new-perspective.md`
2. 在 `SKILL.md` 的视角注册表中新增一行

无需修改其他文件。

## 关键模式

| 模式 | 应用方式 |
|------|---------|
| 专家选择 | 每个视角根据具体输入识别最相关的现实世界专家 |
| 并行智能体 | 所有视角同时运行，提升分析速度 |
| 分层输出 | 为快速浏览者提供执行摘要，为深入研究者提供完整分析 |
| 双语执行 | 内部推理可以密集；面向用户的输出清晰易读 |
| 引用文件架构 | `SKILL.md` 负责编排，详细指令存放在 `references/` 中 |
