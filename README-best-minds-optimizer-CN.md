# Best Minds — 提示词优化器

[README - English](https://github.com/MoGHenry/best-minds-optimizer/blob/main/README-best-minds-optimizer-CN.md)

你向 LLM 提的每个问题，得到的都是泛泛而谈的回答。同样的问题，如果用查理·芒格的思维模型或 Blair Enns 的定价框架重新组织，得到的回答会截然不同——而且好得多。

这个 Agent Skill 自动完成这个改写过程。你提出问题，它找到最适合你这个具体问题的顶级专家，用该专家的思维框架重写你的提示词，然后用通俗易懂的语言给你答案，并附上一个可以立即执行的下一步行动。

## 使用前后对比

### 定价策略

**你输入：** *"我想给我的自由设计工作涨价，但怕失去客户"*

**没有这个技能** — 你会得到公众号式的建议："慢慢涨"、"要自信"。没有框架，没有重新定义问题。

**有了这个技能** — 选定 **Blair Enns**（《Pricing Creativity》作者），将你的问题拆解为 5 个诊断性子问题，然后给出：

> **恐惧几乎总是大于实际风险。** 如果你涨价 30% 并因此失去 20% 的客户，你反而赚了——客户更少，收入更高，管理负担更轻。高价格淘汰的恰恰是那些砍需求、拖付款、消耗你精力的客户。涨价是一道筛选机制，不是冒险。

包含客户流失的数学计算、客户分层框架（先对谁涨）、涨价对话的具体措辞，最后给出行动指引：*"列出你最重要的 5 个客户——涨价 25% 后，你真正会失去哪一个？"*

### 招聘决策

**你输入：** *"我们是一个 15 人的初创公司，刚完成 A 轮融资。该外聘一位工程 VP，还是提拔从创业第一天就跟着我们的资深工程师？"*

**没有这个技能** — 一份中规中矩的优缺点对比。合理，但没有重新定义这个决策。

**有了这个技能** — 选定 **Ben Horowitz**（《创业维艰》作者），重新定义整个问题：

> **核心不是忠诚与能力之争，而是一个可验证的标准：她是否让其他工程师变得更强，还是仍然是那个亲自修最难 bug 的人？** 带 3 个人是资深工程师加管理，带 20 个人以上需要完全不同的能力。

挖掘出大多数创始人忽略的第三个选项（90 天考核期 + 高管教练），警告最差的结果（"给她一个模糊的 VP 头衔，同时偷偷在外面招人"），并给出下一步：*"这周就和她开诚布公地谈一次——告诉她 A 轮阶段 VP 角色的要求，听听她的想法。"*

### 润色你自己写的文字

**你输入：** *"For step 2, MUST breakdown the optimized prompt and then review all available Agent Skills. Check if any of the Agent Skills is helpful for completing the task."*

**有了这个技能** — 识别出这是用户自己写的文本，不是提问。跳过完整的专家流程，快速润色：

> *"After step 1, break down the optimized prompt and review all available Agent Skills. For each skill that matches the task, emit the skill name and a one-sentence reason it was triggered."*

精简措辞、修正语法、改善并列结构——保留你的语气和意图。

### 简单任务

**你输入：** *"读一下这个目录里的 package.json"*

**有了这个技能** — 直接读文件。不找专家，不改写，零摩擦。该技能知道什么时候不该出手。

## 运作原理

每个提示词进入 4 条分流通道，然后路由到对应的处理流程：

| 通道       | 触发条件         | 处理方式                                                                                       |
| :--------- | :--------------- | :--------------------------------------------------------------------------------------------- |
| **跳过**   | 机械任务         | 直接执行 — 不改写，零摩擦                                                                       |
| **润色**   | 用户撰写的文本   | 快速措辞优化 — 精简表达、修正语法、保留语气                                                      |
| **澄清**   | 意图模糊         | 提出选择题获取上下文，然后进入**优化**                                                           |
| **优化**   | 实质性且清晰     | 完整流程：逻辑映射 → 专家选择 → 提示词改写 → 通俗回答 + 具体下一步                                |

**优化**流程分 4 步执行：

| 步骤 | 内容                                                                           |
| :--- | :----------------------------------------------------------------------------- |
| 1    | **逻辑映射** — 问题类型分类：瓶颈 · 资源 · 方向 · 执行 · 权衡 · 诊断            |
| 2    | **专家选择** — 选定具名专家，提取思维模型和关键指标                               |
| 3    | **提示词改写** — 运用专家的框架、术语、盲区覆盖                                  |
| 4    | **通俗易懂的回答** — 3–5 个要点 + 具体的下一步                                   |

**追问不会重新触发流程。** 问"第 3 点能再详细说说吗"会在同一位专家的框架内深入展开，不会重跑整个流程。

**模糊的问题会先澄清** — 通过选择题让你快速回复一个字母，而不用打一大段话。

## 架构设计

该技能采用渐进式加载策略，保持上下文精简。每次调用只加载分流逻辑，详细指令按需加载：

```
best-minds-optimizer/
├── SKILL.md              ← 入口：分流 + 路由（64 行）
└── references/
    ├── optimize.md       ← 完整流程：逻辑映射 → 专家选择
    │                       → 改写 → 输出（225 行）
    ├── clarify.md        ← 澄清流程，然后路由到
    │                       optimize.md（44 行）
    └── polish.md         ← 用户文本的快速润色
                            （35 行）
```

**跳过**只加载 64 行。**润色**加载 99 行。只有完整的**优化**才会加载较大的参考文件。

## 核心差异

**逻辑映射** — 在选择专家之前，先判断问题的结构形态。一个定价问题看起来可能像是"资源"问题（如何分配有限的东西），实际上可能是"方向"问题（该走哪条路）。分类结果决定适用的推理框架：

| 问题类型 | 适用框架 |
|---------|---------|
| 瓶颈 | 约束理论 |
| 资源 | 机会成本 / 资本配置 |
| 方向 | 第一性原理 / 逆向思维 |
| 执行 | 流程设计 / 任务分解 |
| 权衡 | 决策矩阵 / 加权评估 |
| 诊断 | 根因分析 / 5 个为什么 |

**双语执行** — 优化后的提示词使用高密度专业术语，最大化 LLM 的推理质量。你看到的回答则是通俗表达。技术语言在内部驱动更好的思考；清晰的语言是你实际看到的。

**专家组合** — 当问题横跨多个领域时（例如"SaaS 落地页怎么定价和呈现才能提升转化率？"），技能会选择两位互补的专家，将他们的框架融合成一个连贯的回答。

## 输出格式

```
Expert: Blair Enns — 《Pricing Creativity》作者
Logic Structure: Tradeoff / 竞争性利益权衡
Key Metrics: 平均客单价、客户留存率、单客收入

Optimized prompt:
> [LLM 用于推理的高密度技术性提示词]

---

[你实际看到的通俗回答]

Next step: [一个你现在就能执行的具体行动]
```

## 跨领域适用

不只是商业策略。对任何实质性问题，该技能都能找到合适的专家：

- **成年人学钢琴** — Anders Ericsson（刻意练习）
- **电商转化率下降** — Peep Laja（转化率优化）
- **整理车库** — David Allen（系统思维应用于物理空间）
- **睡眠优化** — Matthew Walker（睡眠科学）

只要某个领域有人写过权威著作或做过系统研究，这个技能就能找到他们。

## 安装

```bash
npx skills add https://github.com/MoGHenry/best-minds-optimizer --skill best-minds-optimizer
```

或在 [skills.sh](https://skills.sh) 搜索 `best-minds-optimizer` 安装。
